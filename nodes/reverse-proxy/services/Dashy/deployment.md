# Dashy Deployment Notes (reverse-proxy — Static Build)

Dashy runs as **static files served directly by Caddy** on reverse-proxy. No Node.js runtime is required on this host — the app is pre-built on ai-tools and the `dist` folder is SCP'd across.

> [!NOTE]
> The config has `disableConfiguration: true`, so there is no server-side config editor. Edit `config.yml.tmpl` in the repo and redeploy.

### Layout

| Path | Role |
| :--- | :--- |
| `.../Dashy/config.yml.tmpl` | Tracked Dashy config template (committed to Git) |
| `.../Dashy/dashy.env.example` | Template for secrets |
| `.../Dashy/deploy_dashy.sh` | Renders config from template + reloads Caddy |
| `/srv/dashy/dashy.env` | Real secret values (not in clone) |
| `/opt/dashy/dist/` | Pre-built static files (SCP'd from build host) |
| `/opt/dashy/user-data/conf.yml` | Runtime config (generated by deploy script) |

### Building the dist (on ai-tools)

Only needed when upgrading Dashy or after a fresh setup:

```bash
cd /tmp
git clone --depth 1 https://github.com/lissy93/dashy.git dashy-build
cd dashy-build
npm install
npm run build
```

Then SCP to reverse-proxy:

```bash
ssh root@192.168.50.101 'mkdir -p /opt/dashy'
scp -r /tmp/dashy-build/dist root@192.168.50.101:/opt/dashy/dist
```

Clean up build artefacts:

```bash
rm -rf /tmp/dashy-build
```

### Deploying config changes

1. Edit `config.yml.tmpl` in the repo and push.
2. On reverse-proxy, pull the repo and run the deploy script:
   ```bash
   cd /opt/homelab-repo
   git pull
   sudo /opt/homelab-repo/nodes/reverse-proxy/services/Dashy/deploy_dashy.sh
   ```

### Caddy routing

The `home.{$DOMAIN_NAME}` block in the Caddyfile serves Dashy's static files from `/opt/dashy/dist` with a `try_files` fallback for SPA routing. The config is loaded at `/opt/dashy/user-data/conf.yml`.
